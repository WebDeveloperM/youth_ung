#### 2. **`backend/apps/users/serializers/admin_serializer.py`**

```python
def validate_role(self, value):
    """Можно создавать Admin, Moderator или Coordinator"""
    if value not in [User.ADMIN, User.MODERATOR, User.COORDINATOR]:
        raise serializers.ValidationError("...")
    return value

def validate(self, data):
    """Для координаторов организация обязательна"""
    if data.get('role') == User.COORDINATOR and not data.get('organization'):
        raise serializers.ValidationError({
            "organization": "Для координатора необходимо указать организацию"
        })
    return data
```

#### 3. **`backend/apps/users/views/admin_users.py`**
```python
class IsAdminOrModeratorOrCoordinator(IsAuthenticated):
    """Проверка что пользователь Admin, Moderator или Coordinator"""
    
    def has_permission(self, request, view):
        if not super().has_permission(request, view):
            return False
        return request.user.role in ['Admin', 'Moderator', 'Coordinator']

# Алиас для обратной совместимости
IsAdminOrModerator = IsAdminOrModeratorOrCoordinator
```

#### 4. **`backend/apps/users/views/all_users.py`**
```python
def get_queryset(self):
    """
    Координаторы видят только пользователей своей организации
    Admin/Moderator видят всех
    """
    user = self.request.user
    queryset = User.objects.select_related('organization')
    
    if user.role == User.COORDINATOR:
        if user.organization:
            queryset = queryset.filter(organization=user.organization)
        else:
            queryset = queryset.none()
    
    return queryset.order_by('-date_joined')
```

## 🔧 КАК РАБОТАЕТ:

### Создание координатора:

1. **Админ открывает форму** "Добавить администратора"
2. **Выбирает роль** "Координатор"
3. **Появляется поле "Организация"** (обязательное!)
4. **Выбирает организацию** из выпадающего списка
5. **Заполняет остальные поля** (email, имя, пароль и т.д.)
6. **Нажимает "Создать"**

### Работа координатора:

1. **Координатор входит в систему**
2. **Открывает "Все пользователи"**
3. **Видит ТОЛЬКО** пользователей своей организации
4. **Статистика** показывает данные только по его организации
5. **Не может** редактировать пользователей других организаций

## 📊 ОСОБЕННОСТИ РОЛЕЙ:

| Роль | Организация | Доступ к меню | Видит пользователей |
|------|-------------|---------------|---------------------|
| **Admin** | Опционально | Все меню | Всех пользователей |
| **Moderator** | Опционально | Выбранные меню | Всех пользователей |
| **Coordinator** | **Обязательно** | Не требуется | **Только своей организации** |

### Правила валидации:

- ✅ Админ/Модератор: минимум 1 меню должно быть выбрано
- ✅ Координатор: организация обязательна
- ✅ Координатор: меню не требуется (автоматический доступ к пользователям)

## 📋 API ENDPOINTS:

### Создание администратора/координатора:
```
POST /api/v1/admin/admins/
```

**Body для координатора:**
```json
{
  "email": "coordinator@org.uz",
  "username": "coordinator1",
  "first_name": "Coordinate",
  "last_name": "Test",
  "password": "test123",
  "role": "Coordinator",
  "organization": 2,
  "position": "Coordinator",
  "phone": "+998901234567",
  "allowed_menus": []
}
```

### Получение пользователей (с фильтрацией):
```
GET /api/v1/admin/all-users/
```

**Для координатора:** автоматически фильтруется по organization  
**Для админа/модератора:** возвращаются все пользователи

### Получение организаций:
```
GET /api/v1/organisations/public/organisations/
```

**Response:**
```json
{
  "count": 6,
  "results": [
    {"id": 1, "name": "MGPZ"},
    {"id": 2, "name": "UNG - Ассоциация Молодежи"},
    ...
  ]
}
```
